## 2. 验收前置环境

使用本地环境：

```text
MySQL: 127.0.0.1:3306
Redis: 127.0.0.1:6380
RabbitMQ: 127.0.0.1:5672
App: http://localhost:8080
```

构建方式：

```powershell
$env:MYSQL_PASSWORD="本机 MySQL 密码"
$env:JWT_SECRET="至少 32 位随机字符串"
powershell -ExecutionPolicy Bypass -File scripts\build-local.ps1
```

启动方式：

```powershell
$env:MYSQL_PASSWORD="本机 MySQL 密码"
$env:JWT_SECRET="至少 32 位随机字符串"
java -jar target\campus-activity-1.0.0-SNAPSHOT.jar
```

## 5. 验收中发现的问题

### 5.1 管理员初始化方式已补充

验收脚本通过以下方式临时创建管理员：

```sql
UPDATE app_user SET role='ADMIN' WHERE phone='临时手机号';
```

为避免交付时仍依赖临时 SQL，已新增：

```text
src/main/resources/seed-dev.sql
```

该脚本提供开发管理员：

```text
13900001111 / test123
```

并已验证可通过 PowerShell 管道方式导入。

### 5.2 PowerShell 输出中的中文仍可能乱码

接口实际返回了业务错误信息，但 PowerShell 中部分中文显示为乱码。

判断：

这更可能是 PowerShell 输出编码问题，不一定是接口 JSON 编码问题。

后续建议：

用 Postman、Apifox 或浏览器验证中文响应显示。

### 5.3 数据库已有部分历史乱码数据

活动分类和早期活动数据中仍有乱码。

建议：

后续整理一份干净的初始化数据脚本，重新插入 UTF-8 数据。
