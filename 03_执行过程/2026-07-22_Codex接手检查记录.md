## 4. 当前风险点

### 4.1 Maven 使用 Java 8

问题：

`java` / `javac` 是 Java 25，但 `mvn -v` 显示 Maven 使用 Java 8。

影响：

Spring Boot 3 项目后续构建容易失败。

临时解决：

每次构建前先执行：

```powershell
$env:JAVA_HOME='C:\Program Files\Java\jdk-25'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
```

长期建议：

把系统 `JAVA_HOME` 调整为 JDK 17 或 JDK 25。

### 4.2 配置中存在明文数据库密码

文件：

```text
src/main/resources/application.yml
src/main/resources/application-docker.yml
docker-compose.yml
```

问题：

配置文件中直接写了本地 MySQL root 密码。

影响：

如果后续上传 GitHub，会泄露本地密码。

建议：

后续改成环境变量，例如：

```yaml
password: ${MYSQL_PASSWORD}
```

### 4.3 项目目录被安装包污染

接手时项目根目录包含：

```text
downloads\
erl_crash.dump
```

其中 `downloads\Erlang OTP` 是完整 Erlang 安装目录，文件数量非常多。

影响：

1. 项目体积膨胀。
2. 后续 Git 初始化会非常混乱。
3. `rg --files` 输出被大量无关文件淹没。

建议：

已将 `downloads\` 和 `erl_crash.dump` 移动到：

```text
02_资料与素材\本机运行环境归档
```

并在 `.gitignore` 和 `.dockerignore` 中排除该归档目录。

### 4.4 Redis 当前未启动

端口检查：

应用仍可启动，说明基础接口暂不依赖 Redis 连接立即可用。
