## 简历使用建议

简历中建议只放 GitHub 链接和 3-5 条项目亮点，不要把截图全部放进简历正文。

截图适合用于：

- 面试时展示项目真实性
- 网申附件
- 作品集材料
- 项目复盘

## 当前展示重点

本项目展示重点不是页面效果，而是后端业务链路：

1. JWT 登录鉴权。
2. 管理员活动管理。
3. 用户预约活动。
4. 一人一单限制。
5. 预约码核销。
6. 重复核销拦截。
7. Redis 与 RabbitMQ 作为后端技术栈补充。

## 不建议主动强调的内容

当前版本不建议主动声称：

- 已完成前端页面。
- 已线上部署。
- 已完成高并发秒杀。
- 已完成 Lua 原子扣库存。
- 已完成完整 RBAC 权限系统。

这些内容可以作为后续扩展方向，不要作为已实现能力写进简历。
