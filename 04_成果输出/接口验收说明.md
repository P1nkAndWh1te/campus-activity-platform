## 验收前置条件

需要启动：

```text
MySQL: 127.0.0.1:3306
Redis: 127.0.0.1:6380
RabbitMQ: 127.0.0.1:5672
App: http://localhost:8080
```

构建：

```powershell
$env:MYSQL_PASSWORD="本机 MySQL 密码"
$env:JWT_SECRET="至少 32 位随机字符串"
powershell -ExecutionPolicy Bypass -File scripts\build-local.ps1
```

启动：

```powershell
$env:MYSQL_PASSWORD="本机 MySQL 密码"
$env:JWT_SECRET="至少 32 位随机字符串"
java -jar target\campus-activity-1.0.0-SNAPSHOT.jar
```

运行验收脚本：

```powershell
$env:MYSQL_PASSWORD="本机 MySQL 密码"
powershell -ExecutionPolicy Bypass -File scripts\verify-api.ps1
```

如需重置为干净演示数据：

```powershell
Get-Content -Encoding UTF8 src\main\resources\reset-demo-data.sql | mysql -u root -p --default-character-set=utf8mb4 campus_activity
```

## 验收脚本说明

脚本位置：

```text
scripts\verify-api.ps1
```

脚本会自动：

1. 创建临时普通用户。
2. 创建临时管理员用户。
3. 将临时管理员用户 role 更新为 `ADMIN`。
4. 登录获取用户 token 和管理员 token。
5. 创建测试活动。
6. 预约活动。
7. 验证重复预约被拦截。
8. 核销预约。
9. 验证重复核销被拦截。
10. 创建另一个活动并验证取消预约。
