# 校园活动预约与核销平台 - 简历项目描述

## 一句话定位

校园活动预约与核销平台是一个基于 Spring Boot 3 的 Java 后端项目，面向校园讲座、比赛、社团活动等场景，完成用户登录、活动管理、活动预约、名额扣减、预约码核销、活动详情缓存和预约通知等核心链路。

## 简历推荐写法

### 项目名称

校园活动预约与核销平台

### 项目角色

Java 后端开发

### 技术栈

Spring Boot 3、MyBatis-Plus、MySQL、Redis、RabbitMQ、JWT、BCrypt、Knife4j、Maven、Docker Compose

### 项目描述

基于 Spring Boot 3 开发校园活动预约与核销后端系统，支持用户注册登录、活动分类与活动列表查询、管理员活动管理、用户活动预约、预约取消、管理员预约码查询与核销等功能。项目使用 MySQL 保存核心业务数据，Redis 缓存活动详情，RabbitMQ 异步写入预约通知，并提供 Knife4j 接口文档和 PowerShell 自动化接口验收脚本。

### 项目亮点

1. 设计用户、活动分类、活动、预约、核销记录、通知等核心表结构，通过 `activity_reservation` 表上的 `(user_id, activity_id)` 唯一索引，从数据库层兜底限制同一用户重复预约同一活动。

2. 预约接口使用 `@Transactional` 保证名额扣减和预约记录写入的一致性，并通过 `available_quota > 0` 条件更新完成库存扣减，避免库存被扣成负数。

3. 使用 JWT 完成登录认证，在拦截器中解析 `Authorization: Bearer token`，将用户 ID 和角色写入 `UserContext`，支撑普通用户接口和管理员接口的权限判断。

4. 活动详情采用 Redis Cache-Aside 模式，先查缓存，未命中再查 MySQL 并写入缓存；对不存在的活动写入短 TTL 空值缓存，降低缓存穿透风险，并为正常缓存增加随机 TTL 抖动。

5. 预约成功后在事务提交后发送 RabbitMQ 消息，由消费者异步写入通知记录，避免通知写入阻塞预约主流程。

6. 编写 `scripts/verify-api.ps1` 自动化验收脚本，覆盖注册、登录、JWT 鉴权、管理员创建活动、活动详情查询、预约、重复预约拦截、核销、重复核销拦截和取消预约等核心链路。
